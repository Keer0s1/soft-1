# 🎬 VideoMaker

Сайт для сборки роликов под YouTube: загружаешь сценарий (txt) и промты картинок,
софт озвучивает текст через **Voicer API**, генерирует картинки через **fast-gen API**
и собирает готовый **MP4**, где смена картинок попадает под слова.

## Как это работает

1. Ты загружаешь сценарий `.txt` (и при желании отдельный файл с промтами).
2. Весь текст озвучивается **одним куском** через Voicer (выбираешь голос-шаблон).
3. По каждому промту генерируется картинка через fast-gen (до 5 одновременно).
4. Длительность каждой картинки считается **пропорционально длине её куска текста** —
   когда в озвучке говорится про мусорку, на экране мусорка.
5. ffmpeg склеивает картинки и озвучку в MP4 — скачиваешь готовый ролик.

## Формат сценария

**Вариант 1 — один файл.** После каждого куска текста — строка с промтом
(`IMG:`, `ИЗО:` или `КАРТИНКА:`):

```
Жил-был кот, который любил гулять по дворам.
IMG: рыжий кот гуляет по двору, мультяшный стиль

Однажды он нашёл старую мусорку за домом.
IMG: старая ржавая мусорка за домом, мультяшный стиль
```

**Вариант 2 — два файла.** Сценарий разбит пустыми строками на абзацы,
во втором файле промты по одному на строку: абзац №1 → промт №1 и т.д.

## Установка и запуск

Нужны Python 3.11+ и ffmpeg.

```bash
# ffmpeg (Ubuntu/Debian)
sudo apt install ffmpeg
# ffmpeg (Windows): скачай с ffmpeg.org и добавь в PATH

pip install -r requirements.txt

cp .env.example .env
# заполни .env: пароль, секрет, ключи и URL API

python -m app.main
```

Открой `http://localhost:8000` — вход по паролю из `.env` (`APP_PASSWORD`).

## Настройка `.env`

| Переменная | Что это |
|---|---|
| `APP_PASSWORD` | пароль входа на сайт (сайт виден только тебе) |
| `SECRET_KEY` | случайная строка для подписи сессии |
| `VOICER_API_URL` | базовый URL Voicer API (где лежит `/openapi.json`) |
| `VOICER_API_KEY` | ключ Voicer (заголовок `X-API-Key`) |
| `FASTGEN_API_URL` | URL fast-gen, по умолчанию `https://googler.fast-gen.ai` |
| `FASTGEN_API_KEY` | ключ fast-gen (заголовок `X-API-Key`) |

## Что показывает сайт

- **Баланс Voicer** (1 символ = 1 единица, минимум 500 за задачу) и твои шаблоны голосов.
- **Лимиты и использование fast-gen** (`/api/v5/usage`), список моделей и возможностей.
- **Бейдж «Видео»** — доступна ли генерация видео по твоей текущей подписке fast-gen.
  Сейчас подписка без видео — софт это покажет; когда подключишь видео, бейдж станет зелёным.
- Журнал каждой задачи: статус озвучки, прогресс картинок, ошибки.

## Модели картинок

- **Flow (Google)**: `GEM_PIX_2` (Nano Pro), `IMAGEN_3_5` (Imagen 4), `NARWHAL` (Nano Banana 2)
- **Flower**: Nano Banana 2 (модель одна)

Форматы кадра: 16:9 (обычный YouTube), 9:16 (Shorts), 1:1, 4:3, 3:4.

## Что проверить при первом запуске

1. **URL Voicer API** — в присланной документации не был указан базовый адрес,
   впиши его в `VOICER_API_URL`.
2. **Поля создания TTS-задачи** — софт шлёт `POST /tasks` с телом
   `{"text": ..., "template_id": ...}`. Если Voicer ждёт другие имена полей
   (точная схема `TaskCreateRequest` есть в их `GET /llm.md`), поправь
   `app/voicer.py`, функция `create_task` — это одно место.

## Планы на потом

- Генерация **видео-сцен** вместо статичных картинок (fast-gen `flow/video`,
  `flower/video`) — когда подписка будет с видео. Архитектура уже готова:
  добавится провайдер в `app/fastgen.py` и шаг в `app/pipeline.py`.
- Переходы/кен-бёрнс эффект между кадрами в ffmpeg.
