# 🎬 Faceless Video Generator — Полная инструкция по запуску

Веб-приложение для автоматической генерации YouTube-роликов из сценария:
ты пишешь текст по сценам + промты картинок, а сайт сам:
- озвучивает голосом (через **Voicer API**),
- генерирует картинки (через **fast-gen API**),
- собирает MP4 с эффектами (Ken Burns зум, переходы, субтитры, музыка, цветокор) через **ffmpeg**.

> Работает на любом ПК с Windows/Mac/Linux. Эта инструкция написана так, чтобы повторил даже человек, который видит Node.js впервые. Просто следуй по шагам сверху вниз.

---

## 📋 Что должно стоять на компьютере

1. **Node.js версии 20 или новее** — это среда выполнения JavaScript.
   Скачать: <https://nodejs.org/> → бери LTS версию (зелёная кнопка).
   Проверить установку: открой PowerShell / Terminal и набери:
   ```
   node --version
   npm --version
   ```
   Должны показаться номера версий (например `v20.18.0` и `10.8.2`).

2. **PostgreSQL 14+** — база данных, где хранятся все проекты, сцены, история.
   - Скачать: <https://www.postgresql.org/download/>
   - При установке поставь пароль для пользователя `postgres` — **запомни его**.
   - Также установится **pgAdmin 4** — удобный интерфейс для управления БД.
   - Проверить: открой pgAdmin → должен подключиться к локальному серверу.

3. **Git** (опционально). Скачать: <https://git-scm.com/>

> ⚠️ **ffmpeg ставить НЕ нужно** — подтянется через npm-пакет `ffmpeg-static`.

## 🗄 Создание базы данных

### Вариант 1 — через pgAdmin 4 (для новичков)

1. Открой **pgAdmin 4**.
2. Слева раскрой `Servers → PostgreSQL → Databases`.
3. ПКМ по `Databases` → **Create → Database**.
4. В поле **Database** напиши `soft` → **Save**.

Готово. Таблицы создадутся автоматически на следующем шаге через Prisma.

### Вариант 2 — через SQL-скрипт (если хочешь руками)

В папке `backend/prisma/init.sql` лежит готовый скрипт создания всех таблиц без данных.

```bash
# Создай пустую БД
psql -U postgres -c "CREATE DATABASE soft;"

# Применить скрипт со всеми таблицами
psql -U postgres -d soft -f backend/prisma/init.sql
```

## 🚀 Установка и запуск (пошагово)

### Шаг 1. Скачать проект

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd soft-1
```

Или просто скопируй папку `soft-1` в любое место на диске.

### Шаг 2. Настроить backend

```bash
cd backend
npm install
```

Это скачает все зависимости (~5 минут при первом запуске).

Создай файл `.env` рядом с `package.json` (скопируй с `.env.example`):

**Windows (PowerShell):**
```powershell
Copy-Item .env.example .env
```

**Mac/Linux:**
```bash
cp .env.example .env
```

Открой файл `.env` в любом редакторе и заполни:

```env
# Подключение к БД — подставь свой пароль от postgres
DATABASE_URL="postgresql://postgres:ТВОЙ_ПАРОЛЬ@localhost:5432/soft?schema=public"

# Voicer (озвучка)
VOICER_API_URL="https://voiceapiru.csv666.ru"
VOICER_API_KEY="вставь_сюда_ключ_с_voicer"

# fast-gen (картинки)
FASTGEN_API_URL="https://googler.fast-gen.ai"
FASTGEN_API_KEY="вставь_сюда_ключ_с_fastgen"

# Папка для файлов (картинок, озвучки, видео)
DATA_DIR="./data"
PORT=8000

# Параллельные генерации картинок (макс 5 у fast-gen)
IMAGE_CONCURRENCY=5

# Кодек видео — libx264 (CPU) | h264_nvenc (NVIDIA, в 5x быстрее) | h264_qsv (Intel) | h264_amf (AMD)
VIDEO_CODEC="libx264"
VIDEO_PRESET="veryfast"
VIDEO_FPS=24
```

### Шаг 3. Создать таблицы в БД

```bash
npm run prisma:generate    # сгенерировать Prisma-клиент
npm run db:push            # создать/обновить таблицы в БД
```

Если видишь `Your database is now in sync with your Prisma schema. Done` — всё ок.

### Шаг 4. Запустить backend

```bash
npm run dev
```

Сервер запустится на <http://localhost:8000>. Не закрывай это окно терминала.

### Шаг 5. Запустить frontend (отдельный терминал)

```bash
cd frontend
npm install
npm run dev
```

Frontend стартует на <http://localhost:5173>. Открой эту ссылку в браузере.

## ✅ Проверка что всё работает

1. Открой <http://localhost:5173> — должна загрузиться главная страница со списком проектов.
2. Внизу страницы будет индикатор статуса бекенда (зелёная точка = ОК).
3. Нажми **"+ Новый проект"** → введи название → откроется редактор сцен.
4. Если индикатор красный — проверь:
   - Запущен ли backend (`npm run dev` в `backend/`)?
   - Правильный ли `DATABASE_URL` в `.env`?
   - Запущен ли PostgreSQL (в Windows: Services → найти `postgresql-x64-XX` → должен быть Running)?

## 🔄 Перенос на другой компьютер

Если переносишь проект:

1. **Скопируй всю папку** `soft-1` (или склонируй с GitHub).
2. На новом ПК выполни шаги 1–5 из раздела "Установка".
3. **Папка `data/` не нужна** — там старые проекты, картинки и видео конкретно того ПК.
   Можно скопировать, если хочешь сохранить старые ролики.
4. **БД нужно создать заново** — на новом ПК делаешь `npm run db:push` и получаешь пустую базу.
   Если хочешь перенести данные — экспорт через pgAdmin (Backup) → импорт (Restore).
5. **API-ключи в `.env`** — впиши свои новые (или те же).
6. **node_modules не копируй** — `npm install` сам поставит всё что нужно.

## 🆘 Частые проблемы

### "Cannot connect to database"
- Проверь что PostgreSQL запущен (Windows: `services.msc` → `postgresql-x64-XX`).
- Проверь пароль в `DATABASE_URL` (должен совпадать с тем что ставил при установке Postgres).
- Попробуй подключиться через pgAdmin к серверу с тем же паролем.

### "Port 8000 already in use"
- Кто-то уже занял порт. Поменяй `PORT=8000` на `PORT=8001` в `.env`.
- Не забудь поправить `vite.config.js` (proxy target) если меняешь порт бекенда.

### "Voicer API error 401" / "fast-gen 401"
- Неправильный API-ключ. Проверь что скопировал ключ полностью без пробелов.
- Проверь баланс на сервисе.

### "ffmpeg failed"
- Скорее всего проблема с правами доступа к папке `data/`.
- На Windows: ПКМ по папке проекта → Свойства → Безопасность → проверь что у пользователя есть полный доступ.

### Превью получается странное / эффекты не применяются
- Открой dev-tools браузера (F12) → Console → смотри ошибки.
- В терминале backend смотри логи генерации.
- Удали папку `data/projects/<id>/preview/` — пересоберёт с нуля.

### Команда `psql` не найдена (Windows)
- Добавь путь к PostgreSQL в PATH: `C:\Program Files\PostgreSQL\16\bin` (или твоя версия).
- Или используй pgAdmin вместо `psql`.

## 🎯 Возможности приложения

- **Сценарий по сценам**: текст озвучки + промт картинки на каждую сцену
- **Импорт сценария** из текста (с маркерами `IMG:`) или двух колонок
- **Перетаскивание сцен** (drag-and-drop)
- **Эффекты на сцену**: пресеты зума (наезд/отъезд/панорама/дрифт/дыхание/кино), сила, скорость, плавность (linear/easeIn/easeOut/easeInOut), точка фокуса, дрожание камеры
- **Переходы между сценами**: 30+ xfade-вариантов с настройкой длительности
- **Субтитры**: word-level (караоке) или scene-level, 4 стиля, цвета, обводка, тень, фон, размер
- **Цветокор**: яркость, контраст, насыщенность, температура, LUT-файлы
- **Плёночные эффекты**: зернистость, виньетка
- **Музыка**: фон с auto-ducking (тише когда говорят)
- **SFX**: точечные звуковые эффекты на таймлайне
- **CTA-оверлеи**: подпишись/лайк (текст/эмодзи/картинка/видео)
- **Общие оверлеи**: текст/картинка/видео с анимациями (in/out/idle)
- **Свой голос**: загрузить mp3 вместо AI-озвучки
- **NLE-таймлайн**: визуальный редактор как в видеоредакторах
- **Быстрый пререндер**: low-res превью за секунды (1:1 совпадает с финалом по эффектам)
- **Real-time прогресс**: через WebSocket, без перезагрузки страницы
- **История сборок**: все ролики с настройками сохраняются
- **Архив проектов**: soft-delete с восстановлением
